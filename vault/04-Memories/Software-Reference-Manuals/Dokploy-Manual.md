# Dokploy Reference Manual

**Scope:** Docker Swarm-based container deployment, CLI remote server management, REST API webhook integrations, and template configurations on Linux hosts.

---

## 🛠️ 1. Installation & Environment Override

Dokploy installs as a Docker container, initializing a single-node Docker Swarm cluster to handle routing, health checks, and rolling updates.

### A. Bootstrap Command
```bash
curl -sSL https://dokploy.com/install.sh | sh
```
This script automates:
1.  Checking for Docker and installing it if missing.
2.  Initializing Docker Swarm (`docker swarm init`).
3.  Deploying Dokploy and Traefik on the Swarm overlay network.

### B. Configuration Ports
*   **Port 3000**: Exposes the administrative dashboard.
*   **Port 80/443**: Exposes Traefik proxy ingress paths.

## 💻 2. CLI Tool Usage (`dokploy-cli`)

The CLI interacts with the Dokploy REST API, allowing developers to trigger deployments, query logs, or check build states from terminal pipelines.

### A. Installation & Authentication
```bash
# Global install via npm
npm install -g @dokploy/cli

# Authenticate command line
dokploy auth -u https://dokploy.nobsdash.com -t YOUR_API_TOKEN
```
Alternatively, configure authentication using environmental variables:
```bash
export DOKPLOY_URL="https://dokploy.nobsdash.com"
export DOKPLOY_API_KEY="your-api-key"
```

### B. Common CLI Commands

```bash
# Check CLI status and version
dokploy --version

# List all created projects
dokploy project all

# List all applications under a project
dokploy application all

# Deploy a specific application by UUID
dokploy application deploy --applicationId "app-uuid-here"

# Redeploy a compose stack by UUID
dokploy compose deploy --composeId "compose-uuid-here"

# Fetch logs for a specific service
dokploy application logs --applicationId "app-uuid-here"
```

## 🔌 3. REST API Webhook Integrations

Dokploy exposes an OpenAPI schema. To trigger actions programmatically (such as deploying a patch generated by an AI loop), use curl against the endpoints.

### Authentication Header
Every request requires the header: `x-api-key: <API_KEY_HERE>`.

### A. Deploy an Application (`POST /api/application.deploy`)
*   **Request:**
    ```bash
    curl -X POST "https://dokploy.nobsdash.com/api/application.deploy" \
         -H "x-api-key: your-api-key" \
         -H "Content-Type: application/json" \
         -d '{"applicationId": "app-uuid-12345"}'
    ```
*   **Response:**
    ```json
    {
      "message": "Deployment triggered successfully",
      "deploymentId": "dep-998877"
    }
    ```

### B. Deploy a Docker Compose Stack (`POST /api/compose.deploy`)
*   **Request:**
    ```bash
    curl -X POST "https://dokploy.nobsdash.com/api/compose.deploy" \
         -H "x-api-key: your-api-key" \
         -H "Content-Type: application/json" \
         -d '{"composeId": "compose-uuid-67890"}'
    ```

## 📝 4. Docker Compose Configurations

When deploying custom applications via compose files inside Dokploy, connect them to the native routing overlay network.

### Compose File Example for Dokploy
```yaml
version: '3.8'

services:
  nobs-api:
    image: node:alpine
    container_name: nobs-api
    restart: always
    # Connect container to external Traefik router network
    networks:
      - dokploy-network
    environment:
      - NODE_ENV=production
      - DATABASE_URL=${DATABASE_URL} # Reads variables defined in Dokploy UI

networks:
  dokploy-network:
    external: true
    name: dokploy-network
```
*   **Traefik Subdomain Routing**: Instead of configuring complex Traefik labels manually in the YAML file, configure your routing domain (e.g. `api.nobsdash.com`) directly in the **Domains** tab for that Compose stack in the Dokploy dashboard. Dokploy automatically generates the labels and applies them to the Swarm routing mesh.
